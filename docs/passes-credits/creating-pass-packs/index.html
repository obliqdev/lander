Creating Pass Packs

Last updated April 2026 · 5 min read

What is a pass pack?

A pass pack is a one-time purchase of a set number of class credits. Clients buy the pack upfront and then use credits to book classes or appointments. Unlike memberships, passes do not auto-renew — once all credits are used or the pass expires, clients need to purchase again.

Common use cases: Intro offers (e.g., "3 classes for $30"), drop-in 10-class packs, or appointment bundles for private sessions.

Step 1 — Pack name & description

Navigate to Passes → New Pass.

  1. Enter a Pass Name that clearly communicates the offer (e.g., "Intro 3-Pack", "10-Class Bundle", "Private Session 5-Pack").

  2. Add a Description explaining what the pass covers, who it is for, and any restrictions.

  3. Optionally mark it as an Intro Offer. Intro passes are limited to one per client — a second purchase attempt will be declined and the client will see a message directing them to regular pricing.

Step 2 — Credit count & pricing

  1. Set the Number of Credits included in the pass. Each credit typically corresponds to one class booking, but you can configure some class types to cost more than one credit.

  2. Set the Price. Obliq displays the per-credit value automatically to help clients understand the offer.

  3. Enable Multi-Use if credits can be shared between household members or guests, or leave it disabled to restrict the pass to the purchasing client only.

Tip: For intro offers, pricing 3 classes at roughly 70% of the standard single-class drop-in rate is a common and effective entry point.

Step 3 — Expiry rules

  1. Choose an Expiry Type: Fixed Date (the pass expires on a specific calendar date) or Rolling (expires N days from purchase date).

  2. Set the Expiry Period. For a 10-class pack, a rolling 90-day window is typical. For seasonal packs, use a fixed date.

  3. Toggle Notify Before Expiry to send clients an automated reminder email 7 days before their pass expires. This is a useful retention nudge to purchase another pack.

Step 4 — Restrictions & redemption

  1. Set Class Type Restrictions — which types of classes the pass can be used for. For example, a group class pass might not be redeemable for private appointments.

  2. Set a Credits Per Booking value if certain classes should cost more credits (e.g., specialty workshops cost 2 credits instead of 1).

  3. Set Purchase Limits — the maximum number of times a single client can purchase this pass. Leave blank for unlimited purchases.

Heads up: Once a client redeems credits from a pass, the pass cannot be deleted — only deactivated. Deactivating a pass prevents new purchases but does not affect existing pass balances.

Step 5 — Publish your pass

  1. Set the pass status to Active. Active passes are visible on your booking page and can be purchased by clients.

  2. Optionally set a Sale Period — a start and end date during which the pass is purchasable. This is useful for limited-time intro offers or seasonal promotions.
